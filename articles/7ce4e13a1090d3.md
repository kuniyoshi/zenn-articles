Unity プロジェクトで開発しているとき、UnityEngine.Object の null 比較にヒントが出ているのを見つけました。null 比較がオーバーライドされているからですが、「これってエラーにしないと見逃すリスクの方が大きいのでは?」と思いました。

[Microsoft.Unity.Analyzers](https://github.com/microsoft/Microsoft.Unity.Analyzers?tab=readme-ov-file) に多くの定義があったので全部確認して設定しました。

| ID      | 内容 | 設定値 | コメント |
|---------|------|--------|------|
| UNT0001 | 空の Unity メッセージでもランタイムから呼ばれるため、不要なら削除して無駄な処理を避けるべきです。 | warning | 同意 |
| UNT0002 | タグ比較は `==` より組み込みの `CompareTag` を使うほうが高速です。 | warning | 同意 |
| UNT0003 | `GetComponent` 系 API は型安全性のため、非ジェネリック版よりジェネリック版の使用が推奨されます。 | warning | 同意 |
| UNT0004 | `Update` はフレームレート依存のため、`Time.fixedDeltaTime` ではなく `Time.deltaTime` を使うべきです。 | warning | 同意 |
| UNT0005 | `FixedUpdate` では `Time.deltaTime` ではなく `Time.fixedDeltaTime` を使うべきです。 | warning | 同意 |
| UNT0006 | この Unity メッセージはメソッドシグネチャが正しくありません。 | error | シグネチャを合わせるべき |
| UNT0007 | Unity オブジェクトは null 比較演算子を独自実装しているため、null 合体演算子（`??`）と互換性がありません。 | error | 大体意図しないバクのためエラーです。意図してやりたいときは `if` で書きます。 |
| UNT0008 | Unity オブジェクトは null 比較演算子を独自実装しているため、null 条件演算子（`?.`）と互換性がありません。 | error | 大体意図しないバクのためエラーです。意図してやりたいときは `if` で書きます。 |
| UNT0009 | `InitializeOnLoad` 属性をクラスに付ける場合は、エディタ起動時に呼ばれる静的コンストラクタを用意する必要があります。 | error | コードは整理しましょう |
| UNT0010 | コンポーネントは `new` ではなく `AddComponent()` で作成し、`GameObject` にアタッチする必要があります。 | error | Unity の作法なので従いましょう |
| UNT0011 | `ScriptableObject` は `CreateInstance()` で生成し、Unity メッセージ処理のために Unity 側で正しく生成させるべきです。 | error | Unity の作法なので従いましょう |
| UNT0012 | コルーチンは `StartCoroutine()` で開始しないと実行されません。 | error | 現代では使わなそうです |
| UNT0013 | `SerializeField` は public フィールドでは冗長で、static / readonly フィールドには無効です。 | error | 冗長なのは許可できますが無効なのはエラーにしたいです |
| UNT0014 | `GetComponent` 系 API は `UnityEngine.Component` 継承型、またはインターフェース型に対してのみ使うべきです。 | error | 同意 |
| UNT0015 | `InitializeOnLoadMethod` / `RuntimeInitializeOnLoadMethod` / `DidReloadScripts` を付けたメソッドは static かつ引数なしである必要があります。 | error | 同意 |
| UNT0016 | `Invoke` などで文字列リテラルのメソッド名を渡すのは型安全でないため、`nameof` やコルーチンの直接呼び出しを使うべきです。 | error | あまり使うことはなさそうです |
| UNT0017 | 可能なときは `SetPixels32` を使いましょう。 | warning | やったことないので保留 |
| UNT0018 | `Update` など高頻度メッセージ内で `System.Reflection` を使うと遅くなり、ラグの原因になるため避けるべきです。 | warning | エラーではない |
| UNT0019 | `GameObject.gameObject` は自身を返すため、多重アクセスは冗長で可読性と性能を損ねます。 | error | コードを整理しましょう |
| UNT0020 | `MenuItem` 属性は static メソッドにのみ適用でき、メニューコマンド化する対象は static 関数である必要があります。 | error | エラーですね |
| UNT0021 | Unity メッセージは public である必要がないため `protected` が推奨され、継承時の隠蔽による呼び出し漏れも防ぎやすくなります。 | warning | `private` にすることが多かったですが、継承することもあるかもしれません。個人プロジェクトだと `none` にするかも |
| UNT0022 | `Transform` へのアクセス回数を減らすため、`position` と `rotation` を別々に設定せず `SetPositionAndRotation()` を使うべきです。 | suggestion | あまり出番はないです |
| UNT0023 | Unity オブジェクトは null 比較演算子を独自実装しているため、null 合体代入（`??=`）と互換性がありません。 | error | 大体意図しないバクのためエラーです。意図してやりたいときは `if` で書きます。 |
| UNT0024 | 密なループや性能重視箇所では、ベクトル演算よりスカラー演算のほうが高速です。 | suggestion | あまり出番はないです |
| UNT0025 | `Input.GetKey*` は文字列より `KeyCode` 引数のオーバーロードを使うほうが安全で効率的です。 | error | 良い方を使おう |
| UNT0026 | `TryGetComponent` は対象コンポーネントが存在しない場合にアロケーションが発生しないため、`GetComponent` より効率的です。 | suggestion | 違いが気になる場面はぼぼないので読みやすい書き方でいい |
| UNT0027 | `PropertyDrawer` を継承した場合、`OnGUI(...)` を実装しないとインスペクタに `no GUI implemented` と表示されます。 | warning | 警告ですね |
| UNT0028 | Physics クエリには非アロケーション版があるため、`RaycastAll` などは `*NonAlloc` 系 API への置き換えが推奨されます。 | suggestion | 存在への同線としてだします |
| UNT0029 | Unity オブジェクトは null 比較演算子を独自実装しているため、`null` とのパターンマッチと互換性がありません。 | error | 大体意図しないバクのためエラーです。意図してやりたいときは `if` で書きます。 |
| UNT0030 | `Object.Destroy` / `Object.DestroyImmediate` に `Transform` を渡すのは無効で、実行時エラーになります。 | error | エラーになるので |
| UNT0031 | `InitializeOnLoad` 系の初期化タイミングではアセット読み込み前の場合があるため、アセット操作を行うと null になる可能性があります。 | suggestion | 保留 |
| UNT0032 | Unity 2021.3.11f1 以降では `localPosition` と `localRotation` の個別設定より `SetLocalPositionAndRotation()` の使用が推奨されます。 | suggestion | 個別の方が読みやすそう |
| UNT0033 | この Unity メッセージはメソッド名の大文字・小文字が正しくありません。 | error | バグですね |
| UNT0034 | `Vector3` は `Vector2` に変換でき、その際 `z` 成分は破棄されます。 | none | 意図して書く方が多そう |
| UNT0035 | `Vector2` は `Vector3` に変換でき、その際 `z` 成分は 0 になります。 | none | suggestion でもいいですが UNT034 との生合成のため |
| UNT0036 | `Transform` へのアクセス回数を減らすため、`position` と `rotation` を別々に読むより `GetPositionAndRotation()` を使うべきです。 | suggestion | 個別でいいけど、存在への導線はあってもいい |
| UNT0037 | `Transform` へのアクセス回数を減らすため、`localPosition` と `localRotation` を別々に読むより `GetLocalPositionAndRotation()` を使うべきです。 | suggestion | 個別でいいけど、存在への導線はあってもいい |
| UNT0038 | `WaitForSeconds` などを毎回生成するとメモリ使用量が増えるため、キャッシュして GC 負荷を下げるのが有効です。 | suggestion | あまり使わなそう |
| UNT0039 | `RequireComponent` 属性を使うと必要なコンポーネント依存を自動付与できます。 | none | 自動付与のメリットより、手動除去のデメリットが嫌です。参照は SerializeField でつけるし |
| UNT0040 | `GameObject.isStatic` はエディタ専用で、ビルド後は常に `false` になるためエディタ専用コードで扱うべきです。 | error | バグですね |
| UNT0041 | `Animator` メソッドを繰り返し呼ぶ場合は、毎回ハッシュ計算が走る文字列版より int（ハッシュ）版の利用が効率的です。 | suggestion | 文字列でやってもいい場所での煩さが気になる |
| UNT0042 | `Mesh` の `vertices` など配列プロパティをループ内で都度参照すると毎回新規配列が確保され、特に `Update()` などで性能低下を招きます。 | suggestion | 使う機会がないので保留 |
